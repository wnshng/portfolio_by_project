FAST 원칙 기반으로 얼굴/팔/음성 특징을 정의하고, MLP 판단과 단계형 UI 흐름을 구성한 캡스톤입니다.

좌우 비대칭 기준 설계멀티모달(얼굴/팔/음성)단계형 UI

어떤 식으로 진행했는지

  1. 01

    주제 및 배경 정의 (p.2, p.4)

    FAST 기반 뇌졸중 감지 시스템을 주제로 선정하고, 골든타임 내 전조 증상 인지의 어려움을 문제로 정의함.

  2. 02

    시스템 플로우 제시 (p.6)

    얼굴→팔→음성 순서로 측정하는 흐름을 시스템 플로우로 구성함.

  3. 03

    구현 방식 설계 (p.7)

    MediaPipe Face Mesh(얼굴 468좌표), MediaPipe Pose+Hands(팔/손 좌표), Praat+parselmouth(음성 특징), WebSocket 통신 구조를 사용함.

  4. 04

    얼굴 특징 설계 (p.8)

    Yaw/Roll, 입꼬리 거리·기울기, 좌우 비대칭 좌표를 특징으로 수집하고 MLP로 얼굴 비대칭 여부를 판단함.

  5. 05

    팔 특징 설계 (p.9)

    어깨/팔꿈치/손목+손가락 좌표, 좌우 팔 벡터 각도 차, 손목 깊이·높이 차, 손 방향(손가락 3차원 좌표 포함)으로 MLP를 학습함.

  6. 06

    음성 특징 설계 (p.10)

    Jitter/Shimmer/Intensity를 추출하고 3 Hidden Layer + ReLU + Dropout(40%) 구조로 음성 이상 여부를 판단함.

  7. 07

    실시간 통신 구성 (p.11)

    WebSocket 기반 실시간 양방향 통신 구조를 적용함.

  8. 08

    UI 단계 구성 (p.12)

    얼굴 감지→팔 측정→음성 측정→최종 결과 화면 순서의 단계형 UI를 제시함.

위 단계는 「최종발표.pdf」의 p.2~12 기준으로 정리했습니다.
